GHOST CHAIN: A Technical White Paper

1. Abstract: Ghost Chain is an ephemeral Peer-to-Peer (P2P) protocol that utilizes a "Lighthouse" discovery mechanism to establish encrypted WebRTC tunnels between nodes. Unlike traditional blockchains that prioritize permanent storage, Ghost Chain prioritizes memory-only persistence.

2. The Signaling Layer: Ghost Chain solves NAT traversal using a GitHub Gist as a Beacon (Lighthouse). Nodes post a randomized Peer ID, prefixed with a Room Hash, to a hidden Gist. Every 10 seconds, a scanBeacon operation pulls the latest IDs. A GitHub Action (Janitor) wipes the Gist daily to ensure zero-trace operation.

3. Consensus: Uses a simplified GHOST protocol. Longest Chain Wins: When peers connect, they adopt the longer chain as the "Truth." Mining refers to local block creation broadcast via WebRTC.

4. Security: Room names are Base64 hashed (Unique Frequency). Transport uses mandatory WebRTC DTLS encryption. Application-level secrecy supports AES-GCM encryption via the Secret Key.

5. Persistence: Zero-Knowledge Hosting. Volatile Memory storage (RAM-only). Never written to Disk, LocalStorage, or Cookies.

GHOST CHAIN: USER OPERATING GUIDE

1. Prerequisites: You need the Access URL and a PAT with Gist authorization.

2. Initializing: Select a Room Name (shared frequency) and a Secret Key (encryption). Paste your PAT and click INITIALIZE_GHOST.

3. Communication: Once connected, type data and click MINE. The new block is broadcast directly into your peer's browser RAM.

4. The Kill Switch: Closing the tab or refreshing acting as a master purge, instantly erasing the local ledger.
